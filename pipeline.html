Pipeline

The research workflow is broken into stages. Each script entry links to its repository path.

Corpus Construction

Corpus assembly and initial library export.

Step Script LLM / API Platform Description
1 — — Zotero Literature search across 8 databases (Scopus, WoS, PubMed, IEEE Xplore, ACM DL, JSTOR, ProQuest, Google Scholar). Import to Zotero group library. Attach PDFs. Export as QUAL-AI.bib via Better BibTeX.

Bibliometric Harvesting

Reference harvesting and metadata enrichment from scholarly APIs.

Step Script LLM / API Platform Description
2 harvest_and_enrich.py OpenAlex, Semantic Scholar, Crossref, Unpaywall Local Two-phase pipeline. Phase 1 (harvest): API cascade for cited reference lists — picks whichever API returns most refs per item. Phase 2 (enrich): 7 passes for OpenAlex concepts/keywords, Semantic Scholar abstracts, Unpaywall OA status, GSCC citation counts, reference metadata merging. WoS/Scopus stubs for institutional API activation. Fully resumable.

Metadata Enrichment

Supplementary metadata extraction, classification, and Zotero citation repair.

Step Script LLM / API Platform Description
3a extract_keywords_from_pdfs.py Claude Sonnet + Zotero API Local Downloads PDFs from Zotero cloud. Extracts first 2 pages. Sends to Claude to read author-provided keywords exactly as printed. Flags items where publication has no keywords section (N/A). Updates Kumu Excel.
3b classify_study_type.py Claude Sonnet Local Classifies each corpus item as Empirical/Applied, Methodological/Technical, or Conceptual/Critical from title + abstract + keywords. Returns confidence scores and one-sentence reasoning per item.
3c map_venue_disciplines.py — Local Maps 258 publication venues to 10 discipline categories via deterministic lookup + fuzzy keyword matching. Categories: Health Sciences, Computer Science, Qualitative Methods, Social Sciences, Education, HCI, Business, Communication, Humanities, Interdisciplinary.
3d update_zotero_citations.py OpenAlex, Semantic Scholar, Crossref Local Repairs missing DOIs via Crossref title lookup. Fetches citation counts from OpenAlex and Semantic Scholar. Writes "Citation Count: N (Source)" into Zotero Extra field. Updates items in-place via Zotero Web API.

Bibliometrix Assembly

Preparation of bibliometrix-ready master files and WoS-compatible text exports.

Step Script LLM / API Platform Description
4a build_bibliometrix_master.py — Local Reads QUAL-AI.bib + harvested JSON files. Matches by normalized DOI. Injects cited reference lists as CR fields in BibTeX format (WoS convention: LastnameInitials, Year, Venue, DOI). Brace-depth-aware parser handles Better BibTeX nested case protection. Post-write validation for entry count and brace balance.
4b normalize_cr.py — Local Cleans all CR field values: strips braces, brackets, quotes, HTML entities. De-duplicates reference strings within each item. Inserts empty CR = {} for items without data so bibliometrix sees a known empty value. Brace-depth-aware field extraction.
4c convert_to_wos.py — Local Converts cleaned BibTeX to WoS plaintext format for R bibliometrix import. Maps fields to WoS tags: AU, TI, SO, DE, AB, DI, CR. Preserves Zotero cite key as UT field for traceability. Sidesteps BibTeX structural incompatibilities (unbraced years, double-brace case protection, Windows paths).
4d fix_wos_c1.py — Local Patches WoS plaintext with C1 (affiliation) and RP (reprint address) placeholder fields. Required because bibliometrix crashes on zero-length vector when both are entirely absent. Placeholder values are overwritten with NA in R before analysis.

Corpus Screening

Inclusion and exclusion screening over the assembled corpus.

Step Script LLM / API Platform Description
5 screen_corpus.py Claude Sonnet Local AI-assisted screening of all items against inclusion/exclusion criteria. 7 inclusion types (Applied Research, Methods & Algorithms, Theoretical, Tools & Frameworks, Researcher Perspectives, Reviews & Syntheses, Field Discourse). 6 exclusion types (Traditional Computational, Computational-Only, General AI-Perception, Superficial Mentions, Out of Scope). Per-item decision, confidence, and reasoning trail.

Keyword Normalization

Normalization of author keywords and Kumu-ready tag fields.

Step Script LLM / API Platform Description
6 clean_kumu_keywords.py — Local Deterministic merge rules: expands abbreviations (NLP → Natural Language Processing, LLM → Large Language Models), merges case/plural variants, removes arXiv category tags and off-topic terms. Produces cleaned Kumu Excel with reduced keyword vocabulary.

Concept Induction

Machine-assisted concept discovery and thematic synthesis.

Step Script LLM / API Platform Description
7a run_lloom.py GPT-4o-mini (OpenAI) Local LLooM unsupervised concept induction on full corpus (title + abstract + keywords as composite text). Rate-limited to 10 req/15s to stay under TPM limits. Generates 14 machine-induced concepts with per-document binary scores and placement rationale. Merges concept scores into Kumu as new columns.
7b claude_synthesis.py Claude Sonnet Local Takes LLooM concept clusters (top 60 papers per concept to manage token limits). Sends abstract bundles to Claude with "critical second coder" instructions. Generates 2–3 page thematic narrative synthesis per concept. Outputs markdown files. Inputs to deductive codebook design.

Corpus Validation

Post-clustering validation and manual thematic review support.

Step Script LLM / API Platform Description
8 sort_pdfs_by_theme.py — Local Parses QUAL-AI.bib to map physical PDF file paths. Copies and renames PDFs from Zotero randomized storage into 14 readable thematic folders on local drive. Enables full-text validation via Google NotebookLM and human review before codebook scoring.

Deductive Codebook Scoring

Application of the final deductive codebook across the corpus.

Step Script LLM / API Platform Description
9 score_corpus.py GPT-4o-mini (OpenAI) Local Scores every corpus item against validated 26-point binary codebook. 5 domains: Methodology (6 criteria), Human-AI Dynamics (6), Limitations (4), Ethics (7), Prompting (3). Forces strict JSON output. Automated checkpointing against API timeouts. Calculates Primary Domain for Kumu color-coding. Outputs scored Kumu Excel.

Synchronization

Alignment of Kumu, BibTeX, Zotero, and public-facing facets.

Step Script LLM / API Platform Description
10a pin_zotero_citekeys.py Zotero Web API Local One-time setup: reads Better BibTeX citation keys from QUAL-AI.bib. Matches Zotero items by DOI first, then normalized title+year fallback. Writes "Citation Key: ..." into Extra field. Refuses to guess on conflicts. Dry-run default. Outputs preview, results, conflicts, and error CSVs.
10b sync_kumu_bib_zotero.py Zotero Web API Local Kumu-authoritative three-way sync. Parses BibTeX with library-first parser + fallback. Cleans metadata. Uses Kumu as authority for titles and keywords. Updates Zotero (title, tags, DOI, abstract) via Web API with retry/backoff logic. Produces synced Kumu XLSX, cleaned BibTeX, and audit trail CSVs. Dry-run default.
10c setup_facets.py Zotero Web API Local Creates Zotero collection hierarchies for Kerko faceted search. Creates parent collections (Inclusion Category, Discipline, Study Type) with sub-collections for each value. Assigns items to appropriate sub-collections. Outputs config.toml facet blocks with collection keys.

CR Extraction (LLM)

Fallback extraction of cited references from PDFs when APIs fail.

Step Script LLM / API Platform Description
11 extract_cr_from_pdfs.py Claude Sonnet + Zotero API Local For ~100 items where APIs returned no cited references. Downloads PDFs from Zotero cloud. Extracts last 5 pages. Sends to Claude for structured reference list parsing (authors, title, year, venue, DOI per reference). Outputs JSON matching harvest_and_enrich schema. Supports both Claude and Gemini backends.

Network Construction

Build of Kumu-ready network outputs from the cleaned corpus.

Step Script LLM / API Platform Description
12 build_kumu_network.py — Local Reads QUAL-AI.bib + WoS plaintext. Builds co-citation network entirely in Python (bypasses bibliometrix edge list due to lossy node IDs). Corpus items keyed by Zotero cite key (UT field). External references keyed by 8-char hex hash of normalized CR string. Outputs Kumu elements CSV (nodes with full metadata) and connections CSV (co-citation edges with weights).

Bibliometric Analysis

Downstream analysis in R and bibliometrix.

Step Script LLM / API Platform Description
13 R / bibliometrix — Local (R) Imports bibliometrix_master_wos_fixed.txt into R via convert2df(). Clears placeholder C1/RP fields. Runs biblioAnalysis(), co-citation network extraction, bibliographic coupling, thematic evolution (4 time slices: pre-2023, 2023, 2024, 2025–26), strategic diagrams, and burst detection.

Web Deployment

Files required to deploy the public bibliography and related web assets.

Step Script LLM / API Platform Description
14a config.toml — PythonAnywhere KerkoApp TOML configuration: Zotero library connection, 6 faceted search filters (Study Type, Discipline, Inclusion Category, Keywords, Item Type, Year), navbar links (Index, Project Overview, Kumu Map, Paper), site metadata.
14b __init__.py — PythonAnywhere Modified KerkoApp factory with CSS injection via after_request handler. White background, black text, sunset orange (#FF6B35) hover accents across all links, buttons, badges, and facet filters.
14c logging.py — PythonAnywhere Fixes PythonAnywhere recursion bug: replaces Flask WSGI error stream (ext://flask.logging.wsgi_errors_stream) with sys.stderr in dictConfig handler. Prevents infinite logging loop caused by PythonAnywhere routing stream.write() back through logger.
14d wsgi.py — PythonAnywhere PythonAnywhere WSGI entry point: adds project directory and user site-packages to sys.path, sets working directory via os.chdir() for config file discovery, creates Flask application via KerkoApp factory.

Each stage is separated to make the workflow easier to scan than a single long table.
