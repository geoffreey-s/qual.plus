A A Bibliometric Pipeline for Mapping AI in Qualitative Research

This repository contains the complete replication pipeline for a scoping review of how artificial intelligence (specifically large language models, generative AI, and transformer-based models) is reshaping qualitative research methods.

The pipeline combines traditional bibliometric methods with LLM-assisted metadata extraction, unsupervised concept induction, and deductive codebook scoring to analyze a corpus of XXX papers (2017–2026).

Live outputs:

  • Interactive Bibliography (Kerko) — faceted search across 6 dimensions
  • Co-citation Network Map (Kumu) — interactive network visualization

Replication Pipeline

Step Research Phase Script LLM / API Platform Description
1 Corpus Construction — — Zotero Literature search across 8 databases. Import to Zotero. Export as QUAL-AI.bib via Better BibTeX.
2 Bibliometric Harvesting harvest_and_enrich.py OpenAlex, Semantic Scholar, Crossref Local API cascade for cited references, citation counts, OpenAlex keywords, abstracts, and Unpaywall OA status.
3a Metadata Enrichment extract_keywords_from_pdfs.py Claude Sonnet Local Extracts author keywords from PDF first pages via Zotero cloud + Claude. Flags items with no keywords section.
3b Metadata Enrichment classify_study_type.py Claude Sonnet Local Classifies items as Empirical/Applied, Methodological/Technical, or Conceptual/Critical from abstracts.
3c Metadata Enrichment map_venue_disciplines.py — Local Maps 258 venues to 10 discipline categories via deterministic rules + fuzzy matching.
4 Corpus Screening screen_corpus.py Claude Sonnet Local AI-assisted screening against 7 inclusion and 6 exclusion criteria. Per-item decision, confidence, and reasoning.
5 Keyword Normalization clean_kumu_keywords.py — Local Deterministic merge rules: expand abbreviations, merge case/plural variants, remove arXiv category tags.
6a Concept Induction run_lloom.py GPT-4o-mini Local LLooM unsupervised concept induction on full corpus. 14 concepts with per-document scores.
6b Thematic Synthesis claude_synthesis.py Claude Sonnet Local Narrative synthesis per LLooM cluster. "Critical second coder" approach. Inputs to codebook design.
7 Deductive Codebook Scoring score_corpus.py GPT-4o-mini Local 26-point binary codebook across 5 domains (Methodology, Dynamics, Limitations, Ethics, Prompting). Primary Domain assignment.
8a Synchronization pin_zotero_citekeys.py Zotero Web API Local One-time setup: pins Better BibTeX citation keys into Zotero Extra field for reliable cross-platform matching.
8b Synchronization sync_kumu_bib_zotero.py Zotero Web API Local Kumu-authoritative three-way sync: Kumu → BibTeX → Zotero. Dry-run default. Full audit trail.
8c Synchronization setup_facets.py Zotero Web API Local Creates Zotero collection hierarchies for Kerko faceted search (Inclusion, Discipline, Study Type).
9 Cited Reference Extraction extract_cr_from_pdfs.py Claude Sonnet Local LLM extraction of reference lists from PDF last pages for ~100 items where APIs returned nothing.
10 Network Construction build_cocitation_network.py — Local Builds co-citation and bibliographic coupling edge lists. Outputs Kumu-formatted network data.
11 Web Deployment Kerko config — PythonAnywhere KerkoApp with custom CSS, 6 facets, WSGI configuration, PythonAnywhere logging fixes.

Technology Stack

Component Technology Role
Reference management Zotero + Better BibTeX Corpus construction, PDF storage, metadata
Bibliometric APIs OpenAlex, Semantic Scholar, Crossref, Unpaywall Cited references, citation counts, keywords, OA status
LLM (structured extraction) Claude Sonnet 4 (Anthropic) Keyword extraction, study type classification, screening, CR extraction, thematic synthesis
LLM (concept induction) GPT-4o-mini (OpenAI) via LLooM Unsupervised concept clustering, codebook scoring
Network visualization Kumu Interactive co-citation and bibliographic coupling maps
Web bibliography KerkoApp (Flask) on PythonAnywhere Public faceted search interface
Data authority Kumu Excel → Zotero → BibTeX Three-way synchronization pipeline
